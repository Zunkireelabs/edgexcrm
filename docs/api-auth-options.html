🟠 Without API Key (Quick Start)
// Works immediately — no setup await fetch('/api/public/submit/admizz/scholarship', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ ... }) }); // Rate limit: 20 submissions per 10 min
🟢 With API Key (Production)
// Higher limits + audit trail await fetch('/api/public/submit/admizz/scholarship', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer crm_live_...', }, body: JSON.stringify({ ... }) }); // Rate limit: 120 submissions per 1 min
  • Zero friction to start — devs can test instantly
  • Upgrade path: add key when going to production
  • API key gives higher rate limits + audit
  • Same endpoint handles both — simple integration
  • Revoke bad actors' keys without breaking others
  • Slightly more complex endpoint logic
  • Anonymous path still vulnerable to abuse
