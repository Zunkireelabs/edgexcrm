API Layer — What Happens on Save
Every save triggers a PATCH request through auth, validation, and finally writes to Supabase.
PATCH /api/v1/form-configs/[id] ● Sending…
Body: { "name": "Scholarship Application", "steps": […], "is_active": true }
AUTH authenticateRequest() → requireAdmin() ✓ owner role confirmed
Reads session cookie → verifies JWT → checks tenant_users role ∈ {owner, admin}
VALIDATE validateFormConfig() + slug uniqueness ✓ slug is unique
Checks: name not empty · slug format (a-z, 0-9, -) · slug unique per tenant · steps not empty
WRITE createServiceClient() → form_configs UPDATE ✓ 200 OK
UPDATE form_configs SET steps = $1, branding = $2 WHERE id = $3 AND tenant_id = auth.tenantId
