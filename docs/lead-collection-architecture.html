Lead Collection Architecture

How leads flow from external websites into EdgeX CRM

1 — Source

External Website / Iframe Embed

Client websites embed the CRM form via iframe or users visit the form URL directly.

2 — Widget Route Group

(widget) Layout — Lightweight, Embeddable

Separate route group with transparent background, no fonts/theme. Designed for iframe embedding.

Server Component (SSG + ISR)

  • Static generation via generateStaticParams()
  • Revalidates every 1 hour
  • Calls getFormConfigByTenantSlug(slug, formSlug)
  • Fetches form_configs + tenants data
  • Passes config as props to PublicForm

Two URL Patterns

  • /form/[slug]/[formSlug] — specific form
  • /form/[slug] — default (first active) form

Layout Features

  • Transparent background
  • DNS prefetch to Supabase
  • No Toaster, no fonts, no theme
▼
3 — Form Rendering

<PublicForm /> — Multi-Step Dynamic Form

Client component (817 LOC) that renders form steps from form_configs.steps JSONB.

Client State

currentStep — which step user is on (0-indexed) formData — { field_name: value } for all text/select inputs fileData — { field_name: File } for file inputs sessionId — generated once per visit (links all partial saves) leadId — set after first save (used for updates) errors — { field_name: error_message } submitted — shows thank-you screen when true

Supported Field Types

text email tel select file textarea checkbox radio date number entity_select

Multi-Step Flow

STEP 1
Personal Info
name, email, phone
→
STEP 2
Details
course, intake, etc.
→
STEP N
Documents
files, terms

Each "Next" click → validateStep() → savePartial() → POST /api/v1/leads

4 — Submission

Form Submission (Partial + Final)

Partial saves on each step, final submit on last step with file uploads.

Partial Save (each "Next")

// POST /api/v1/leads { tenant_id: "uuid", session_id: "abc123def", step: 1, is_final: false, idempotency_key: "{sessionId}-step-1", first_name: "John", email: "john@example.com", ... }

Final Submit (last step)

// POST /api/v1/leads { tenant_id: "uuid", session_id: "abc123def", step: 3, is_final: true, status: "new", idempotency_key: "{sessionId}-final", lead_id: "uuid (from partial)", entity_id: "uuid", form_config_id: "uuid", phone: "+977-9841234567", custom_fields: { ... }, file_urls: { ... } }

Create Path (no lead_id)

  • INSERT INTO leads (...)
  • Handle 23505 (duplicate idempotency)
  • Returns 201 Created

Update Path (has lead_id + session_id)

  • Verify lead exists with matching session
  • UPDATE leads SET ... WHERE id = ?
  • Returns 200 OK

6 — Database Storage

Supabase (PostgreSQL + Storage)

Lead data stored in PostgreSQL, files in Supabase Storage bucket.

leads

id (PK, UUID) tenant_id (FK) pipeline_id (FK) stage_id (FK) entity_id (FK) form_config_id (FK) session_id, step, is_final first_name, last_name email, phone status, assigned_to custom_fields (JSONB) file_urls (JSONB) idempotency_key (UNIQUE) deleted_at (soft delete)

form_configs

id (PK, UUID) tenant_id (FK) name, slug is_active steps (JSONB) branding (JSONB) redirect_url

pipeline_stages

id (PK, UUID) pipeline_id (FK) name, slug position, color is_terminal

audit_logs + events

tenant_id (FK) action / type entity_type, entity_id changes / payload (JSONB) ip_address, user_agent
